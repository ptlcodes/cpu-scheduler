Project Overview

This simulator was built for the Operating Systems CA2 project (Section 2EO46). The goal was to make CPU scheduling algorithms easier to understand by visualizing them interactively.

Users can enter custom processes with arrival times, burst times, and priority values, pick any of 6 algorithms, and instantly see a Gantt chart and performance metrics like average waiting time, turnaround time, and CPU utilization. There's also a comparison page that benchmarks all algorithms on the same workload.

Module Breakdown

Module 01
Input & GUI
Process entry form, algorithm selector cards, time quantum input for Round Robin, and the process queue table. Built in index.html.
Module 02
Scheduling Engine
All 6 algorithm implementations — FCFS, SJF, SRTF, Round Robin, Priority (preemptive + non-preemptive). Lives entirely in algorithms.js.
Module 03
Visualization
Gantt chart renderer, metric cards, per-process stats table, and the side-by-side bar chart comparison in compare.html.

Algorithms Implemented

NameTypeHow it works
FCFS Non-preemptive Runs processes in the order they arrive. Simple but can cause the convoy effect when a long job blocks shorter ones.
SJF Non-preemptive Picks the process with the smallest burst time from the ready queue each time the CPU is free. Gives the lowest average waiting time for a static job set.
SRTF Preemptive Preemptive version of SJF. If a new process arrives with a shorter remaining time than the current one, it preempts immediately.
Round Robin Preemptive Each process gets a fixed time quantum in circular order. Good for interactive systems. Performance depends heavily on quantum size.
Priority NP Non-preemptive The process with the lowest priority number runs to completion before the next one is selected. Can cause starvation of low-priority processes.
Priority P Preemptive If a process with a higher priority arrives while another is running, it immediately takes over the CPU.

How to Use

  • Open index.html directly in any browser — no server needed
  • Select a scheduling algorithm from the cards
  • Add processes manually or click "Load sample" for demo data
  • Hit "Run Simulation" to generate the Gantt chart and stats
  • Go to Compare tab to see all algorithms on the same dataset
